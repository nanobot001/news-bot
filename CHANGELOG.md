# Changelog

All notable changes to this project will be documented in this file.

## [Unreleased]

- **YouTube RSS Ingestion Stabilization & RSSHub Fallback**: Implemented a fallback parser in the local RSSHub instance (`youtubei` parser) that handles channels lacking a standard "Videos" tab. Wrapped standard `getVideos()` calls in a try-catch block and fell back to `getShorts()` when a channel only has shorts/reel items. Mapped and formatted Shorts (`ReelItem` components) to standard RSS fields (video ID, title, and thumbnails) using a minification-safe `isShorts` check. Integrated local RSSHub fallback URL routing (`RSSHUB_BASE_URL` in `.env`) when direct YouTube feeds fail during scheduled ingestion. (2026-05-27)
- **Curation Stabilization & Bulk Keyword Management**: Upgraded `/keyword add` and `/keyword remove` commands to support bulk operations via comma-separated input strings while preserving exact singular backward-compatible outputs. Refactored configuration loader to automatically split and clean comma-separated keyword entries during startup. Implemented robust fallback lookup (`findArticleByMessageOrEmbed`) for articles and DB self-healing. Protected `startThread` operations with try-catch blocks in manual merges and automated polling similarity pipelines to gracefully handle `MessageExistingThread` errors. Unified "Split from Thread" nomenclature to "Remove from Thread" across commands and tests. (2026-05-27)
- **Manual Article Removal & Feedback Loop**: Implemented the `"Remove Article"` context menu command, allowing Bot Managers to retract mistaken posts with a removal reason. Updated the `/stats` and `/audit` commands to track manual removals and output a "Culprit Keywords Summary". Included interactive modals and strict database integrity checks to ensure status transitions only happen upon successful Discord deletion. (2026-05-26)
- **Toronto Eats Sources & Regional News Mass Expansion**: Added 22 new feeds covering regional areas (Markham, Richmond Hill, Scarborough, Etobicoke, Mississauga) and YouTube food & walking channels, bringing total active sources to 82. Implemented exponential backoff retries with connection close headers to prevent CDN socket dropouts. Refined scoring logic to only award location keyword bonuses when at least one core topic keyword is matched. (2026-05-26)
- **Block 2-5 (Topic Keyword Management & Rescoring)**: Added Negative Keyword / Blocked Term management to the `/keyword` slash command with `add-blocked`, `remove-blocked`, and `view-blocked` options, complete with autocomplete and audit logging. Extended the `/refresh` command with a dynamic `lookback` parameter for rescoring and reprocessing unposted articles. (2026-05-26)
- **Block 2-4 (Topic & Source Management Commands)**: Added comprehensive control plane commands for managing topic configurations and RSS feed sources directly from Discord. Implemented administrative command suites `/topic` (list, view, create, set-channel, set-threshold, set-emoji, disable) and `/source` (list, add, remove). Secured command gates using `isBotManager` permission controls. Implemented atomic configuration writes and dynamic reloading to ensure instant, zero-restart in-memory application of settings. Supported skip behavior for disabled topics during polling. Created a dedicated unit/integration test suite in `tests/topicSource.test.ts` and updated `tests/bot.test.ts` and `package.json` configurations. (2026-05-26)
- **Block 2-3 (Bot Manager Authorization & Curation Audit Logs)**: Added SQLite schema for `CurationLog` and updated the news polling job to write detailed curation logs capturing evaluation status, scores, and reason breakdowns. Implemented `isBotManager` authorization gate restricted to `BOT_MANAGER_USER_IDS`, `BOT_MANAGER_ROLE_IDS`, or falling back to `ManageGuild` permission. Implemented `/audit` slash command with topic, limit, query, and status filters, featuring auto-complete for topics and downloadable `.txt` file attachment output for larger results. Added comprehensive unit tests in `tests/audit.test.ts` and updated `tests/bot.test.ts`. (2026-05-26)
- **Block 2-2 (Reaction Email Forwarding)**: Added SQLite schema for `EmailForward` and updated `Article` model. Implemented email service using `nodemailer` with a zero-config test fallback using ephemeral `Ethereal Email` accounts in development and an `ALLOW_ETHEREAL_FALLBACK` toggle. Integrated reaction-based forwarding listener in `src/bot/reactionListener.ts` to trigger on mail/envelope reactions (like `📧`, `✉️`, `📩`, etc.). Added direct message (DM) notifications to reacting users with delivery status (including Ethereal links) or failure details, as well as inline Discord message reaction acknowledgements (`✅` on success, `❌` on failure). Created a suite of unit and integration tests in `tests/forwarding.test.ts` verifying delivery mechanics, database persistence, and idempotency check paths. (2026-05-26)
- **Block 2-1 (Heart Reaction Favorites & Instapaper Sync)**: Added SQLite schema for `UserFavorite` and updated `Article` model with Discord message IDs. Implemented `/favorites` recall slash command with topic, query, source, and relative timeframe filters. Implemented optional Instapaper Simple API integration for syncing favorites, and registered Discord reaction listener to capture heart emojis on posted embeds. Added two-way favorite removal via reaction removal (un-hearting) and the `/unfavorite` slash command with real-time autocomplete suggestions. Added unit testing suite in `tests/favorites.test.ts` and expanded command coverage in `tests/bot.test.ts`. (2026-05-25)

## [0.1.0] - 2026-05-23

### Added
- **Additional Slash Commands**: Registered and implemented `/refresh` (force-refresh feeds), `/stats` (database record metrics), `/search` (article database queries), `/topics` (active configurations list), and `/sources` (feed credentials and configurations list).
- **Expanded Command Test Coverage**: Added mock interaction unit tests in `tests/bot.test.ts` for database stats, command authorization routes, and article keyword searching.
- **Phase 2 Block Specifications**: Added specifications for curation enhancements (`block-2-1` through `block-2-5`) to define admin gates, trust levels, throttling cooldowns, similarity clustering, and daily digests.
- **Block 08 (README Setup & MVP Verification)**: Completed installation, configuration, database setup, execution, and verification documentation. Added detailed manual smoke testing guidelines.
- **Block 07 (MVP Commands)**: Registered `/testfeed`, `/lastposts`, and `/reload-config` slash commands. Implemented in-memory configuration reloading, query helper for recently posted articles, and startup guild command auto-registration.

- **Block 06 (Scheduled Polling)**: Pipeline integration runner using node-cron. Features feed failure tolerance, human-readable structured logging with conditional verbosity, and test suites with global fetch mocks.
- **Block 05 (Discord Embeds)**: Formatting and channel publishing logic using Discord.js. Supports rich embeds with conditional score/debug footers in development mode, automated mock test suite, and a live manual runner.
- **Block 04 (Scoring & Filtering)**: Deterministic keyword scoring, topic eligibility thresholds, trusted source boosts, and blocked term filtering.
- **Block 03 (SQLite Dedupe Storage)**: Prisma SQLite storage layer and canonical URL/title hashing deduplication logic.
- Initial project scaffold.
